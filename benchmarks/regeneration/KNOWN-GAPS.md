# Known gaps in the regeneration benchmark

This file captures methodological debt **before** anyone runs `score.py`
in earnest. The scaffold is committable and the deterministic half
(`run.py`) is usable for shakedown runs, but every item below must be
addressed before a headline number is published.

Surfaced by a three-agent consult on 2026-05-18 (prior-art research +
methodology critique + codebase lookup).

## Prior art to cite in any writeup

This work is **derivative-with-a-real-delta**, not novel. The
strip-and-regenerate frame already exists.

- **Generate README Eval** (patched.codes, 2024) — strips and
  regenerates READMEs from 198 repos; uses BLEU/ROUGE/cosine plus
  structural similarity; Gemini-1.5 SOTA 33.43 vs oracle 56.79.
  https://www.patched.codes/blog/evaluating-code-to-readme-generation-using-llms
- **FActScore** (Min et al., EMNLP 2023) — canonical atomic-fact
  decomposition + retrieval-grounded verification.
  https://aclanthology.org/2023.emnlp-main.741/
- **DocAgent paper** (Yang et al., ACL Demo 2025) — function-level
  docstring generation with a 3-axis rubric (Completeness, Helpfulness,
  Truthfulness). https://aclanthology.org/2025.acl-demo.44/
- **LessLeak-Bench** (2025) — methodology for measuring training-set
  leakage across 83 SE benchmarks; relevant because our corpus is
  almost certainly memorized. https://arxiv.org/html/2502.06215v1
- **CoReQA** (2025) — repo-level QA from GitHub issues; closest
  neighbor for source-as-arbiter. https://hf.co/papers/2501.03447

Our delta: **citation-grounded scoring tied to verifiable line ranges**
+ multi-artifact DAG (README + AGENTS + CLAUDE + llms.txt + API ref +
how-tos as a coordinated set), not function-doc granularity.

## Things to fix before scoring

### 1. Train-set contamination is near-total

All seven corpus repos (`tinydb`, `click`, `typer`, `httpx`, `rich`,
`zod`, `hono`) are top-decile-popularity OSS. Their READMEs are in
every major model's pretraining corpus. The judge isn't reading the
original with fresh eyes — it's pattern-matching against memorized
text. Worse, DocAgent itself may regurgitate memorized phrasing,
inflating coverage Jaccard without ever consulting the source.

**Mitigation:** add at least one post-model-cutoff repo or a
private/synthetic repo as a contamination control before publishing.
LessLeak-Bench's protocol is a reasonable template.

### 2. Single-judge same-family scoring is a vanity metric

`divergence_resolution` is the proposed "killer chart". If
`claude-opus-4-7` adjudicates disputes where DocAgent's output was
generated by an Anthropic backend, that's the same model family voting
for itself with the citation excerpt as a thumb on the scale.

**Mitigation:** wire at least one cross-family judge (Gemini, GPT-4o)
via LiteLLM. Report inter-judge disagreement, not just the Anthropic
mean. Cohen's / Krippendorff kappa expected on a human-spot-check set
of ≥30 divergences before any headline number.

Preference-leakage paper:
https://howiehwong.github.io/preference_leakage.pdf

### 3. `verifier_pass` is necessary but not sufficient

`docagent verify` confirms that `<!-- ground: path:l-l -->` citations
point to **extant line ranges**. It does **not** check that the span
supports the claim. Reporting `verifier_pass` as a quality metric
alongside semantic ones overstates what it proves.

**Mitigation:** keep `verifier_pass` as a gate / pre-filter; never use
it as a headline quality number. Sample N citations independently and
have a judge rate whether the span *supports* the claim.

### 4. Sphinx exclusion missing in `strip_docs`

`DOC_TARGETS_DIRS = ("docs",)` (`run.py:47`) strips the whole `docs/`
tree. Click's `docs/` is Sphinx RST with `conf.py` + autodoc;
`rich`'s `docs/` mixes Sphinx + auto-generated API output. DocAgent
writes Markdown (`docs/reference/<module>.md`, `docs/how-to/<slug>.md`)
— it categorically cannot reproduce RST cross-references.

**Mitigation:** enforce a "skip the directory if `docs/conf.py` exists
or `docs/` contains any `*.rst` file" rule in `strip_docs`.

**Confirmed live on tinydb (2026-05-18 shakedown).** Plan had assumed
tinydb had no `docs/` — wrong: it has `docs/api.rst`, `docs/conf.py`,
`docs/index.rst`, etc. The harness stripped them all and scored
DocAgent's Markdown `docs/reference/*.md` against Sphinx RST sources.
This bites tinydb too, not just click/rich.

### 5. No baselines (identity ceiling, empty floor) — HARNESS LANDED 2026-05-18

`score.py --baseline=identity` and `score.py --baseline=empty` now exist
and run through the same scoring pipeline with degenerate inputs.
**Still need to actually run them** with a real judge model + key
before publishing any number; the harness is ready, the calls have not
been spent.

Without baselines, headline numbers are uninterpretable. Specifically:

- **Identity ceiling** — score original vs. original with the same
  judge. This gives the noise floor; anything within that band is
  judge variance, not signal.
- **Empty floor** — score `""` vs. original. Coverage = 0,
  divergence = N/A; this is where the y-axis starts.
- **No-DocAgent baseline** — raw `gpt-4o-mini` with codebase, no DAG
  scaffold. Quantifies what the agent adds over plain prompting.
- **README-only-from-docstrings** baseline — ablate the agent's
  symbol-index input to isolate the artifact-DAG contribution.

**Sequencing:** run identity + empty as the very first judge calls.
If identity-vs-identity scores below ~0.7 Jaccard, the judge is too
noisy to use as-is and the metric needs tightening (or a panel of
3 judges with majority vote) before live scoring.

### 6. No length normalization → silent-terseness win condition — LANDED 2026-05-18

`JudgePassMetrics.claims_per_1000_tokens` is computed alongside raw
claim counts in `score.py` (approx 4 chars/token). Pair it with
`total_claims` in any chart.

### 7. Adopt FActScore atomic decomposition — LANDED 2026-05-18

`prompts/factual_divergence.md` rewritten to do atomic per-claim
verdicts (`supported / unsupported / contradicted`). `score.py`
consumes the new schema and surfaces `factscore =
supported_count / total_claims`. Citations from prior-art: Min et al.
EMNLP 2023; DeepEval faithfulness pattern.

### 8. Adopt DocAgent paper's 3-axis rubric — LANDED 2026-05-18

Three new prompts (`prompts/rubric_completeness.md`,
`prompts/rubric_helpfulness.md`, `prompts/rubric_truthfulness.md`)
score the three axes independently 0–5 with anchored bucket
boundaries. Truthfulness consumes the FActScore JSON to avoid
re-paying for claim verification. Pattern from Yang et al., ACL Demo
2025.

## Shakedown findings (2026-05-18, tinydb)

First end-to-end run: `python -m benchmarks.regeneration.run --only
tinydb --backend agent_sdk --max-cost 1.00`.

`results/tinydb-0428e3375e96/run.json`:

```json
{
  "name": "tinydb",
  "sha": "0428e3375e961a5f9f6c00d96d6750de8eb66de8",
  "backend": "agent_sdk",
  "artifacts_written": ["docs"],
  "docagent_init_exit": 0,
  "docagent_verify_exit": 1,
  "cost_usd": 0.008,
  "wall_seconds": 1103.85,
  "stripped_paths": ["README.rst", "docs/"]
}
```

Harness wiring works. Three things to investigate before declaring the
shakedown clean:

1. **Only `docs/` regenerated; no `README.md`, `AGENTS.md`, `CLAUDE.md`,
   `llms.txt` on disk.** v1.0 tinydb live test (HANDOFF.md:106) produced
   README at $0.006 / 117s. This run produced no root files at $0.008 /
   1103s. Likely caused by the v1.0.1 gate-failure-surfacing patch
   (commit 83abb21) silently not-writing root artifacts whose citation
   gates failed against the stripped-clone state — but the full stdout
   was truncated to last 80 lines, so we can't confirm yet. **Fix:**
   re-run with full stdout captured to a file (not `| tail -80`), and
   trace which artifacts hit `FAIL X (gate): not written`.
2. **`verify_exit: 1`.** Expected if root artifacts didn't land, since
   `docagent verify` then has nothing to verify or fails on missing
   files. But could also indicate a real citation gate failure on the
   `docs/how-to/*.md` / `docs/reference/*.md` files that did write.
   Separate the two cases.
3. **10x wall-clock regression vs. v1.0 live test.** 1103s vs. 117s.
   Possible causes: agent_sdk's per-tool overhead, full 7-artifact run
   vs. README-only, or interaction with the `docs/` stripping.
   Out of scope for this benchmark but a real signal worth filing.

Cost-from-stdout parse worked (captured $0.008). Backend default fix
worked. Strip+copy mechanics worked. `clones/` and `results/` correctly
gitignored.
