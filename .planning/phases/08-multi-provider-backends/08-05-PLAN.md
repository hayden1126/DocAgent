Retry wiring delta (the only code change in this plan):

```python
# Inside the loop body, replacing the bare AuthenticationError-only wrap:
try:
    response = completion(**completion_kwargs)
except litellm.AuthenticationError as exc:
    raise BackendUnavailableError(
        f"LiteLLM authentication failed for model {self.model!r}. "
        f"Ensure the appropriate API key env var is set (GEMINI_API_KEY, "
        f"OPENROUTER_API_KEY, or ANTHROPIC_API_KEY)."
    ) from exc
except litellm.RateLimitError:
    # One bounded retry. Spike code lets this bubble; production wants
    # graceful handling of transient throttling.
    import time as _time
    _time.sleep(2)
    response = completion(**completion_kwargs)   # second attempt; if THIS raises, let it bubble
# Don't catch BadRequestError — bad input is not transient; re-raise.
```

Note: do NOT add a generic `except Exception` here. Specific exceptions
only. The outer call site (orchestrator) already handles unexpected
exceptions by recording them in `run.findings`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `litellm.RateLimitError` → DocAgent retry decision | The retry budget is fixed at 1; cannot be tuned by a hostile provider |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-12 | DoS | Provider rate-limit storm causing retry-then-fail loop | mitigate | The retry budget is exactly 1, hardcoded. A persistent rate-limit re-raises on the second attempt and the orchestrator records the failure in `run.findings`. No exponential backoff (would invite scope creep into a retry library); no unbounded retries. |
| T-08-13 | Repudiation | Failed retry obscures the original error | mitigate | The second-attempt exception bubbles as-is; the original RateLimitError isn't chained (intentional — chaining a transient RateLimitError into a possibly-different second-attempt error would confuse debugging). If the second attempt succeeds, the original is silently consumed (acceptable — that's the point of retry). |
</threat_model>
