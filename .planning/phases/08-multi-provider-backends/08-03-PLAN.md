versions are a one-constant edit. CONTEXT.md also locks Tier 1 of the
pricing ladder requires the OpenRouter opt-in â€” the shim Tier 1 helper
only fires if the response carries `usage.cost`, which only happens if
the call passed `extra_body={"usage": {"include": True}}`.

Output: The backend now identifies unsupported models with a one-time
WARN, computes per-call cost via the shim, accumulates across turns,
and attaches the total to `GenerationResponse.cost_usd`. The
orchestrator wiring (consuming `cost_usd` in `BudgetTracker.add()`)
lands in 08-04.

```
_TESTED_MODELS: frozenset[str] = frozenset({
    "gemini/gemini-2.5-pro",
    "gemini/gemini-2.5-flash",
    "openrouter/anthropic/claude-sonnet-4-6",
    "openrouter/anthropic/claude-opus-4-7",
    "anthropic/claude-sonnet-4-6",
    "anthropic/claude-opus-4-7",
})

_warned_allowlist_models: set[str] = set()

def _warn_unsupported_model(model: str) -> None:
    if model in _TESTED_MODELS:
        return
    if model in _warned_allowlist_models:
        return
    _log.warning(
        "[unsupported-model] %r is not on the tested-model allowlist for "
        "DocAgent (v1). Generation will proceed but citation quality is "
        "not guaranteed. See README 'Multi-provider setup' for the "
        "current allowlist.",
        model,
    )
    _warned_allowlist_models.add(model)
```

Wiring inside `LiteLLMBackend.run()` (deltas only â€” keep everything else from Plan 08-01):
1. Immediately after the lazy `import litellm; from litellm import completion`
   block, call `_warn_unsupported_model(self.model)`. Placement matters:
   AFTER import (so it doesn't fire when `litellm` isn't installed â€”
   that's a different failure mode handled by BackendUnavailableError)
   but BEFORE the first `completion(...)` call.
2. Introduce a local `accumulated_cost: float = 0.0` at the start of
   the loop bookkeeping (next to `input_tokens = 0`, `output_tokens = 0`).
3. Build the `completion(...)` kwargs as a dict and conditionally
   inject `extra_body={"usage": {"include": True}}` when
   `self.model.startswith("openrouter/")`. Then call
   `completion(**kwargs)`. RESEARCH.md Pitfall 2 confirms BOTH
   `extra_body={"usage":{"include":true}}` AND top-level
   `usage={"include":true}` work â€” prefer `extra_body` form for
   cross-provider compatibility (most providers ignore unknown
   `extra_body` keys; some reject unknown top-level kwargs).
4. After each turn's `response = completion(...)` (and the existing
   token-accumulation lines), import the pricing shim lazily and
   add per-turn cost:
       from docagent.backends._litellm_pricing import cost_for_response
       accumulated_cost += cost_for_response(self.model, response)
   The shim is import-safe without litellm; placing the import inside
   run() keeps the module top import-safe too.
5. Replace the return statement to thread cost through:
       return GenerationResponse(
           content=content,
           tool_calls=tool_calls_total,
           input_tokens=input_tokens,
           output_tokens=output_tokens,
           cost_usd=accumulated_cost,
       )
