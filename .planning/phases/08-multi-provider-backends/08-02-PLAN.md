<objective>
Implement the LiteLLM pricing shim with the three-tier fallback ladder
locked in CONTEXT.md and RESEARCH.md Pattern 2 / Code Example 3.

Purpose: Phase 5's `pricing.estimate_cost()` is hand-maintained for
Anthropic-via-SDK only. The LiteLLM path needs a different pricing source
because LiteLLM speaks 100+ providers and DocAgent can't hand-maintain
every one. CONTEXT.md locks: "delegate to `litellm.completion_cost(response)`."
Empirical research found this raises bare `Exception` for unmapped routes
(e.g. `openrouter/anthropic/claude-sonnet-4-5`) — the shim MUST catch
broad `Exception`, not just `BadRequestError`. Pitfall 1 documents the
exception-class drift. Tier 1 prefers OpenRouter's authoritative
server-reported `usage.cost` field (when present) over LiteLLM's
upstream static table; Pitfall 2 explains why.

Output: A pure-Python shim with no side effects beyond logging, a
fresh module-private `_warned_pricing_models: set[str]` (NOT extending
Phase 5's `_warned_models` — kept semantically separate per RESEARCH.md
"Pattern 2 Option A" recommendation), and the `cost_usd` field on
`GenerationResponse` so later plans can thread per-call cost through
the orchestrator without changing the `BudgetTracker` signature
incompatibly.
</objective>

Three-tier ladder (verbatim from RESEARCH.md Code Example 3):
```
def cost_for_response(model: str, response: Any) -> float:
    # Tier 1: OpenRouter server-reported cost
    if model.startswith("openrouter/"):
        server_cost = _openrouter_server_cost(response)
        if server_cost is not None:
            return server_cost
    # Tier 2: litellm.completion_cost
    try:
        import litellm
        return float(litellm.completion_cost(completion_response=response))
    except Exception as exc:
        if model not in _warned_pricing_models:
            _log.warning(
                "litellm could not price model %r (%s); recording $0.00 for this call. "
                "Token counts unaffected. See README troubleshooting for details.",
                model, exc.__class__.__name__,
            )
            _warned_pricing_models.add(model)
        return 0.0

def _openrouter_server_cost(response: Any) -> float | None:
    usage = getattr(response, "usage", None)
    if usage is None:
        return None
    cost = getattr(usage, "cost", None)
    if cost is None:
        return None
    try:
        return float(cost)
    except (TypeError, ValueError):
        return None
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Tier 1 input → response.usage.cost | OpenRouter-supplied cost field crosses into BudgetTracker; could be a malicious string |
| Tier 2 path → litellm.completion_cost | LiteLLM upstream price table maintained by third-party; could shift values silently |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-03 | Tampering | OpenRouter `response.usage.cost` value | mitigate | `_openrouter_server_cost()` coerces via `float()` inside a `try/except (TypeError, ValueError)` block — any malformed value falls through to Tier 2 rather than poisoning the cost-add path. Tests pin the bad-value case. |
| T-08-04 | Repudiation | Unknown-model pricing silently returning 0.0 | mitigate | The Tier 3 path emits a stderr WARN with model name + exception class. The dedup ensures it's visible (one line per model) without log spam. Users see the WARN; CI captures stderr. |
| T-08-05 | Denial of Service | Catastrophic exception inside `litellm.completion_cost` | mitigate | Broad `except Exception` (intentional, per Pitfall 1) catches every failure mode including bare Exception, BadRequestError, RuntimeError. Function never raises; orchestrator path stays alive. |
</threat_model>
